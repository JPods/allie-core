## Notes on This First Week

**Why these items:**

C-W20-1 (hilly terrain kink): The fix we applied today (zeroing Z) is correct for
flat terrain. Hilly terrain is the real test. High enough uncertainty to be a genuine
WhatIf; close enough to the code that it will be observable soon.

C-W20-2 (station looping): Bill flagged this weeks ago and it has been deferred.
The deferred item plus the animation architecture suggests this is an animation
consumption problem, not a routing problem. Posting it forces a check.

C-W20-3 (Alice promotion gate): The pattern recognition loop was designed with a
promotion step that requires Bill's activation. That step requires someone to run it.
If no session has explicitly done so, the loop is open but the gate is stuck. Observable
by checking alice_log for items older than 30 days with status: candidate.

C-W20-4 (CLAUDE.md gaps): Every document has assumptions invisible to its author.
The cold-read test will happen eventually; this item forces the check at 4 weeks.

A-W20-1 (F-07 student exposure): Feature list items without owners and dates tend
to be discovered by users before they are fixed. F-07 has no owner and no date.

A-W20-2 (wcapi concurrent write): This is Allie's own blind spot — she has never
been run in parallel with another Allie instance during an active Alice write session.
The risk is architectural, not behavioral.

A-W20-3 (bill.md timing): A prediction about the wisdom layer's own value. If this
is wrong (technical successors read it first), the layer is doing better than expected.

A-W20-4 (headway drift): WI-004 in whatif.md identified this concern. Posting it
here forces a specific check with a specific date.
