## Why These Items

**C-W22-1 (trip_id format mismatch):** The booking response was built to return `nora_id` but the index.html wrappers normalize to `vehicle_id`. One side of that bridge has not been tested against live Ruby code. Mismatches at response field names are the most common first-run failure in new API wiring.

**C-W22-2 (natalie_dispatch_idle not tested):** The function adds to `@@pods` mid-animation — a code path that has never been exercised. Adding to a live animation array while the timer is ticking is likely to have an off-by-one or missing-field issue that only appears at runtime.

**C-W22-3 (camera follow jitter in U-turns):** The 0.25s tick rate was chosen without measurement. U-turn geometry has tight curvature; the camera's look-at target can move sharply between ticks at station approach speed. This is a feel issue, not a crash — worth noting before demo conditions.

**C-W22-4 (station names shift vocabulary):** The feature's real value is reducing cognitive load in multi-station networks. High confidence because Bill already thinks in place-name terms; S### is a tool artifact.

**C-W22-5 (self.class pattern elsewhere):** The bug was fixed in `dispatch_server.rb` mount_proc blocks. Other module proc patterns exist in the plugin (console callback wrappers, observer blocks). Low confidence because those paths have been tested, but the pattern is architecturally endemic.

**C-W22-6 (station_line_end re-tagging):** The model was modified today (May 27, 19:15) but the scan found zero gw_* entities. Only one explanation: the tags were never applied to the geometry. The prior lines.json was a world-space scan from a live network (S005 in station_test.skp), not from the template model itself.

**C-W22-7 (math as standard for fixed templates):** Every fixed-geometry template hit the same problem this session — the scanner walks solid edges, returns wrong lengths, misses direction. Math is exact; scanning is approximate. The principle was established clearly enough that it should become explicit policy.

**C-W22-8 (Noelle suffix handling):** Noelle's build code looks up cpu segment names by exact string match. The _0/_1 suffix is a new convention established this session. Without a code change to handle the suffix, multi-CPU templates will build with wrong segment wiring.

**C-W22-9 (PROX_TOL_MM elsewhere):** The traffic_circle7 connector arcs (1000mm) are shorter than PROX_TOL_MM (1500mm) because they are transition arcs between the main circle arc segments and the spoke segments. Any formation with tight transition geometry will hit the same problem. The scanner's tolerance is tuned for beam-to-beam gaps, not for tight arc transitions.
