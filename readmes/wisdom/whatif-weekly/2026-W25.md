## Why These Items

**C-W25-1 (station_parking Shuffle passes first try):** The code path is proven on two
other templates. The only risk is a lines.json configuration mistake in station_parking,
but the configuration was reviewed and is correct. High confidence.

**C-W25-2 (path_too_short = frame-of-reference):** The fault fires specifically when
advancing platform slots in a network-placed station (not template). The template works.
The difference is the instance transform. `_platform_pts_entry_first` was written assuming
template-local space; network stations need world space. Classic axiom 10 violation.

**C-W25-3 (pts direction surfaces again):** The pass_chains track list is used in at
least 3 places (transit intercept, Show Route, network dispatch). All three concatenate
pts without checking direction. The transit fix only covers one path.

**C-W25-4 (2_thru_dip headway):** Template tests run with a single station instance at
identity transform, so approach distances are minimal. Real network segments add 20–80m
of guideway between stations. Headway math may not scale without adjustment.

**C-W25-5 (Commerce Clause):** Low confidence, but the question is whether the Court's
2026 term produces anything useful. Worth watching — the constitutional argument is central
to JPods' deployment strategy.
