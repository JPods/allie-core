## What changed

Old format: raw XML with sequential node IDs (`SW1103`, `ST111`), no Structure/CP objects.
New format: same XML graph + a `<StructureMeta>` JSON block encoding Structure and
ConnectionPoint objects — required for the browser GUI's interactive editing.

The converter (`route_time/convert_jpd.py`) rebuilds the network topology from scratch,
using the old `<Groups>` metadata (station templates and circle definitions) to infer
centers, headings, and arm orientations. The old raw nodes and lines are discarded;
new nodes with the correct naming convention are generated by `build_station()` and
`build_traffic_circle()`.

### How to check a file

1. Start the Route-Time server:
   ```bash
   cd /Users/williamjames/Documents/08_JPods/03_Technology
   python3 -m route_time.gui
   ```
2. Open `http://localhost:5000` in a browser
3. Load a converted file: File → Load → navigate to `/Applications/RouteTime_JPods/converted/`
4. The network loads locked (read-only). Click `🔓 Edit` to inspect CPs.
5. Click on a CP (colored dot at a station or circle arm) — it should highlight.
6. Click a second CP — a guideway pair should connect them (green line drawn).
7. Shift-click the connected CP to disconnect.

### What to look for

| Check | Expected | Flag if |
|-------|----------|---------|
| Stations appear on map | Yes, at correct lat/lon | Missing or wildly wrong position |
| Circles appear on map | Yes, 4 arms visible | Missing arms or wrong orientation |
| CP dots appear after unlock | Yes, colored dots at each arm end | No dots → StructureMeta not loading |
| Connection lines | Exist between connected structures | Lines crossing through unrelated structures |
| Station heading | NB guideway roughly pointing "up" the network | Station rotated 90° off |
| Circle arm alignment | Arms roughly align with connecting guideways | Arms pointed into empty space |

### Programmatic check (faster for batch)

```python
from route_time.io.jpd_reader import load
from route_time.engine.network import vincenty_m
import os

src = "/Applications/RouteTime_JPods/converted"
for fname in sorted(os.listdir(src)):
    if not fname.endswith(".jpd"):
        continue
    path = os.path.join(src, fname)
    net, structs, cps = load(path)
    n_st  = sum(1 for s in structs if s["structure_type"]=="station")
    n_tc  = sum(1 for s in structs if s["structure_type"]=="traffic_circle")
    n_con = sum(1 for c in cps if c.get("connected_to")) // 2
    n_open = sum(1 for c in cps if not c.get("connected_to"))
    
    # Flag anomalies
    flags = []
    if len(structs) == 0:
        flags.append("NO STRUCTURES")
    for c in cps:
        if c.get("connected_to"):
            for p in cps:
                if p["cp_id"] == c["connected_to"]:
                    d = vincenty_m(c["center_lat"], c["center_lon"],
                                   p["center_lat"], p["center_lon"])
                    if d > 5000:
                        flags.append(f"LONG_CONN {c['cp_id']} {d:.0f}m")
                    break
    
    status = "OK" if not flags else "WARN: " + "; ".join(flags)
    print(f"{fname}: {n_st}st {n_tc}tc {n_con}conn {n_open}open — {status}")
```

### Known acceptable patterns

- **Open CPs on outer edges** — normal; outer arms of a network have no external connection
- **Short connections (< 30 m)** — adjacent circles or circle-adjacent-to-station; correct
- **0 connected pairs** — single-station sketch files; correct
- **All circles arms=[90,0,270,180]** — this is E/N/W/S; correct for all files where
  the C1-C4 nodes sit exactly east/north/west/south of the circle center (the majority)

### Known limitations to report back to Claude Code

If you see these, flag them with the filename and describe what's wrong:

1. **Station heading off by 90°**: the NB guideway goes east-west but should go north-south
   (or vice versa). File a note: `heading wrong: {filename}, expected ~{N}°, got ~{E}°`

2. **Circle arms misaligned**: the CP dots on the circle arm don't align with the incoming
   guideway. File a note: `circle arm off: {filename}, circle ID, arm index, expected heading`

3. **Connection to wrong structure**: a line from station A goes to circle C when it should
   go to circle B (visible as a guideway that crosses through the wrong area of the map).
   File a note: `wrong connection: {filename}, from {cp_id} to {connected_to}, expected partner`

4. **Structures missing**: a known station or circle from the source .jpd doesn't appear
   in the converted file. File a note: `missing structure: {filename}, old ID`
