## Lessons for Allie

1. **Intersection routing: one algorithm for all templates** — Sally's `_final_approach_tracks`
   finds the first track shared between the hold loop and landing_chain. Returns partial loop
   to that branch point + chain tail to platform. Every fixed-geometry template uses this.
   New templates: populate `landing_chains` in lines.json, let Sally find the intersection.
   Do not author `to_platform` manually unless intersection routing cannot work (no shared track).
   Domain: [SketchUp]

2. **Module boundary check before adding a method** — `_pts_tail_from_near` was placed in
   `module JPodVehicleAnim` (lines 664–2395) but the caller was in `module RubyNatalie`
   (lines 54–496). Wrong module AND wrong namespace. Cost: one reload cycle and a crash.
   Rule: find the `module`/`class` boundary above the call site before adding the method.
   The enclosing module is the right home, not the file.
   Domain: [SketchUp]

3. **Ruby constants cannot be assigned inside method bodies** — `LIFT_RE = /gw_lift/i`
   inside a `def` raises `SyntaxError: dynamic constant assignment`. Plugin fails to load
   entirely — no partial failure, no warning. Rule: regex patterns used inside a single method
   are local variables (`lift_re`), not constants. Constants belong at module level.
   Domain: [SketchUp]

4. **`_pts_tail_from_near`: clip polyline to forward tail** — Projects a near_pt onto each
   segment of a polyline (standard closest-point-on-segment formula), returns the insertion
   point + all vertices forward from there. Never reverses. Correct approach for any
   "pod is somewhere on this track, continue forward" problem. Added to `RubyNatalie`
   alongside `build_maneuver_from_tracks`; invoked via `clip_start: true` parameter.
   Domain: [SketchUp]

5. **Trip panel as a reusable UI primitive** — `showTripSequence(text, label)` was built
   generically from the start: a labeled `<div>`, optional label param, `_TRIP_PANEL_TASK_IDS`
   list, clears on task nav. This is how console panels should be added — not replacing
   output but persisting alongside controls. The same div will serve vehicle trip display
   when that feature is built.
   Domain: [SketchUp]

## Scars

**Module placement** — `_pts_tail_from_near` added to `JPodVehicleAnim` but called from
`RubyNatalie`. Both the module and the namespace were wrong. Cost: one reload cycle and a
crash. The fix was unambiguous once the module boundary was read (line 54 vs line 664).
Check the enclosing `module` line before writing a helper — context in the editor can be
misleading if you're scrolled to the middle of a long file.

**LIFT_RE in method body** — Adding a regex constant inside a `def` raises `SyntaxError`
and kills the entire plugin load. The error message names the file and line but the user
sees only "Extension errors" in SketchUp. No partial degradation — total load failure.
Rule: if it's used only inside one method, it's a local variable.
