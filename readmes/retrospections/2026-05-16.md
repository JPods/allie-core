## Lessons for Allie

1. **v2 followme.json platform location** — Platforms live at `raw['models'][station_id]['platforms']`, not `raw['platforms']`. The `structure_id` is the dict key, not a field in the record. Inject it on read. Domain: [SketchUp] [MeshMobility]

2. **`ensure_platform_host_guideway` requires start_m + end_m** — This method synthesizes a 2-point host guideway from platform endpoints. If `start_m`/`end_m` are nil (because the exporter dropped them), it returns nil silently. The proximity fallback then picks the nearest guideway by geometry — which is often the wrong one (e.g. a 0.946m internal U-turn track). The symptom is `platform_host: false` and `path_total_m < 1.0` in the occupancy conflict diagnostic. Domain: [SketchUp]

3. **Silent format failures accumulate** — Four distinct silent failures can chain: wrong JSON key → empty platforms → no station_ids → wrong guideway → occupancy conflict. Each step returns gracefully. The real error is never surfaced until a vehicle physically overlaps. Check format assumptions explicitly at load time, not at use time. Domain: [SketchUp] [Cross-domain]

4. **`platform_in` is a valid parking surface, `platform_out` is not** — The station model has two guideways: `platform_in` (where vehicles wait before departure) and `platform_out` (exit-only). Only `platform_in` and `platform` are valid parking surfaces. `platform_out` must NOT be in `PLATFORM_TAGS`. Domain: [SketchUp]

### Lessons for Allie

9. **Bezier junction kink: waypoint insertion doesn't work — composite does** —
   Inserting a synthetic waypoint 2m from the stub tip forces the spline through that point,
   but the Catmull-Rom tangent at the waypoint doesn't match the stub direction → kink.
   The correct approach: build the bezier BETWEEN the 2m-offset points; prepend/append
   the 2m straight sections separately. Both sides of the junction are then co-directional
   by construction, not by approximation. Domain: [SketchUp] [Cross-domain]

10. **Sort contamination: never sort synthetic points with user markers** —
    `bezier_spline_pts` sorts markers by projection onto the FROM→TO axis.
    When synthetic lead points were passed into the same array, they were reordered
    by the sort (especially when a station faces back toward the other station),
    producing 90° zigzags. Synthetic points must always be inserted AFTER the sort.
    Domain: [SketchUp]

11. **Show FollowMe: `stub_pair` tag in `track_entity?`** — stub_pair entities have
    profile/cross-section face geometry that `trace_polyline_from_edges` traces as a
    centerline. This produced spurious cross-section lines in the overlay. Remove
    `tag == "stub_pair"` from `track_entity?` to suppress them. Domain: [SketchUp]
