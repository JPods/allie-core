### 5. Arduino Watchdog Root Cause — Missing Keepalive

**Discovery:** During bench motor tests, motor stopped after exactly 1 second with no I2C error. Root cause: `oneSecEvent()` in Arduino firmware checks `conCount == prevConCount` → sets `runFlag=false`. `conCount` only increments on I2C reads, not writes. A bench script that only calls `motor.setSpeed()` (write) sends zero reads → watchdog fires at 1s.

**Fix:** Add `motor.read()` every ~0.9s in bench scripts as keepalive. Production loop is safe (reads every cycle).

No Arduino reflash needed for this; it's a Python bench script issue.

### 7. Motor Ramp Tests — Both Pods

Commanded ramp: 140 → 240 → 140 mm/sec (3 mm/sec steps, 0.9s dwell). `motor.read()` called each step for keepalive + actual speed display.

**Key findings:**
- Below ~156 mm/sec: `actual` holds at 142 (PID floor on free-spinning wheels — no load)
- Above ~156 mm/sec: `actual` tracks `cmd` linearly ✅
- Peak: commanded 240 → actual 227–241 ✅
- Occasional `actual=255` at high speed ramp-down: encoder noise spike in 5-sample average, non-fatal
- No I2C errors on either pod during full ramp
- Quantized steps: 142, 156, 170, 184, 198, 213, 227, 241 (~14 mm/sec per step) — normal, from averaging window
