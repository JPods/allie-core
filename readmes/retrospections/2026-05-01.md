# Retrospection — 2026-05-01

## What Changed

- Implemented stop-and-dump strategy for 5V vehicle placement: on any occupancy conflict the test
  aborts immediately, prints a full diagnostic table to Ruby Console (slot, platform, guideway, vehicle
  world pos, conflict complaint, function + call chain), and pops a UI.messagebox alert.
- Removed the backoff/shift-and-retry mechanism entirely. Was masking real bugs.
- Fixed `vehicle_path_for` for platform host guideways: added override in `jpod_platform.rb` that
  skips `stitch_structure_followme_paths` when `platform_host=true`. The stitcher was matching the
  synthetic 2-point guideway's endpoints against station internal FollowMe paths and appending 14
  extra points, collapsing a 23.9 m path to 0.422 m and forcing `t=1.0` for every slot.
- Fixed `distance_to` → `distance` in `jpod_conflict_detector.rb` (SketchUp Ruby API has no
  `distance_to`). Also corrected unit conversion from `/ 0.3048` (feet→m) to `/ 1.m`
  (SketchUp internal inches → m).
- Fixed false personal-space conflict: threshold lowered from `< 3.0` to `< 2.5` m. Slots are
  designed to be 3 m apart; floating-point gave 2.9999…m which crossed the old threshold.
- Fixed false merge conflict: added `!own_segment.empty?` guard. Freshly placed vehicles have no
  `current_line_id`; empty string matched every vehicle to every other.
- 5V test now places vehicles correctly across platform slots with spreading t values (0.063, 0.188…).

## Root Cause / Lesson

- **Stop-and-dump beats backoff loops.** Three sessions of backoff shuffling produced zero placed
  vehicles and no actionable information. One run with a hard stop and full variable dump exposed four
  distinct bugs simultaneously.
- **`stitch_structure_followme_paths` is greedy.** It will match any guideway endpoint that lands
  near a station's internal FollowMe terminus, including synthetic platform host guideways. Any
  future synthetic guideway that should NOT be extended must override `vehicle_path_for` or set a
  guard attribute.
- **Unit conversions in SketchUp**: SketchUp internal unit is inches. `/ 1.m` converts to meters.
  `/ 0.3048` converts feet to meters — wrong here. Check unit assumptions whenever a distance
  comparison produces results that seem off by ~3×.
- **Conflict thresholds must respect designed spacing.** A personal-space threshold equal to the
  designed slot spacing will fire on every adjacent pair. Threshold should be the minimum physical
  overlap distance (vehicle body length / 2), not the desired spacing.
- **Empty string is a valid match in Ruby `==`.** Any equality check on optional attributes must
  guard against the empty/nil case before comparing.

## Files Modified

- `jpod_vehicle_runtime.rb` — stop-and-dump on conflict; removed backoff accumulator; hard-stop loop
- `jpod_platform.rb` — `vehicle_path_for` override to skip stitching for platform host guideways
- `jpod_conflict_detector.rb` — `distance_to`→`distance`, unit fix, threshold 3.0→2.5, empty-id guard

## Session 2 — May 1, 2026 (afternoon)

### What changed

**1. Vehicle headway rules** (`jpod_guideway.rb`)
- Rule 1: hard stop propagates to all same-path followers when leader is stopped or gap < 0.3 m
- Rule 2: same-path vehicles sorted leader-first before tick — no phantom passing
- Rule 3: gap regulation ±0.1 m/tick on non-platform guideways when gap < 3 m (PERSONAL_ZONE_DIST). Platform guideways exempt.
- Added `GAP_ADJUST_PER_TICK = 0.1.m`, `is_platform` flag stored in `@@anim_state`, `trip_path_cache` (shared path object_id required for grouping).

**2. Console role selector** (`dialogs/console.html`)
- `<select id="role-select">` in header. Default: Operator.
- `ROLE_CATEGORIES` maps roles to category strings from `TASKS` registry.
- `renderSidebar(role)` filters sidebar; fires on `change` event and on `initTasks`.

| Role | Categories |
|---|---|
| Operator | Vehicles, Animation, Nora |
| Builder | Network, Noelle |
| Tester | Network Check, Safety, Noelle |
| Advanced | All |

**3. Console vs Network Editor design rule**
- Network Editor = document editor with viewport (spatial, map-authoring)
- Console = command panel with parameters (execution, reporting)
- Rule: shows something in viewport or edits JSON → Editor. Takes params and returns status → Console.

**4. Menu collapsed to single entry point** (`main.rb`)
- `Extensions > JPods` = one item. Opens Network Editor + Console simultaneously. No submenus.

**5. 13 tasks migrated into Console** (`jpod_console.rb`)
- Place Structure, Place Markers, Deploy Guideway, Import Terrain, Add Structure to Guideway, Connect Gates, Recompute CPs, Clear Overlays, Clear All Markers → Network (Builder)
- List Structures & CPs, Check Dead-End Caps → Network Check (Tester)
- Clear All Vehicles → Vehicles (Operator)
- Run Noelle + Natalie Oversight → Noelle (Builder + Tester)

**6. Reload Plugin button** added to Console header (`cmd_reload_plugin` callback).

**7. Duplicates removed** — Start/Stop Animation, Place Vehicles, Run 5V Test, List Platform Tagged Items, Show FollowMe, Export Bundle each now have exactly one home.
