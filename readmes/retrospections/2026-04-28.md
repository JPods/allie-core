## Processor Knowledge Librarian System (third task, same session)

### What was built

Bill's instruction: "Allie needs to keep a log of what she learns in each role so she can pass along
her rules, understandings, and how she came to them to future processors. She needs to define the tools
that Noelle, Natalie, and Nora processors will use when Allie is not running all of them. She needs a
means to harvest what those processors experience so she becomes their librarian."

| File / Directory | What it does |
|-----------------|-------------|
| `readmes/processor-knowledge/noelle.md` | Noelle knowledge: rules, 3 understandings (with provenance), processor contract, experience log protocol, open questions |
| `readmes/processor-knowledge/natalie.md` | Same for Natalie — 4 understandings, Dijkstra inputs/outputs contract, diag_grid sanity check |
| `readmes/processor-knowledge/nora.md` | Same for Nora — 5 understandings, physical vs simulation separation, timing constants |
| `readmes/processor-knowledge/README.md` | Library protocol: how Allie reads/adds to these files; how future processors use and write back |
| `logs/processor-experiences/noelle-log.jsonl` | Stub file — where future standalone Noelle processor writes experience |
| `logs/processor-experiences/natalie-log.jsonl` | Same for Natalie |
| `logs/processor-experiences/nora-log.jsonl` | Same for Nora |
| `scripts/allie-harvest-processors.py` | Harvest script: reads experience logs → calls deepseek-r1:8b to identify lesson candidates → Allie promotes confirmed lessons to knowledge files |

### The Library Loop

```
Allie acts as Noelle/Natalie/Nora → learns → writes to processor-knowledge/<agent>.md
Future processor starts → reads knowledge file → inherits all accumulated understanding
Future processor operates → writes to logs/processor-experiences/<agent>-log.jsonl
Allie runs allie-harvest-processors.py → LLM identifies lesson candidates
Allie promotes confirmed lessons → knowledge file grows → next processor inherits more
```

### Key design decision

Knowledge files use Rule → Understanding → Provenance format — not just what, but HOW Allie came
to know it. This lets future processors (or a future Allie with no session memory) evaluate whether
a lesson still applies, rather than blindly inheriting conclusions.
