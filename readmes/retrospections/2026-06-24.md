## Lessons for Allie

1. **500mm gaps are edge hallucinations** — this keeps coming back because compression destroys the lesson. 250mm each side of centerline. Any gap 350-650mm is measuring the wrong thing. Axiom 10. [Cross-domain]
2. **Silent tolerance produces failures only experience can diagnose** — and experience gets compressed. Every defect must be visible to the user without discovery. The flag system is the infrastructure for this. [Cross-domain]
3. **Revert and ship beats elegant and broken** — terrain raycast "proper fix" (Entities.raytest) broke Build. The working bounding-box skip was reverted in 5 minutes. Don't hold working code hostage to better architecture. [Cross-domain]
4. **Sally's 3-step direct transform is the right pattern** — no animation maneuvers for conveyor. Sally owns her platform. Direct entity moves, immediate, no clock dependency. [SketchUp]
5. **Natalie's dispatch registry extends to zipper merge** — same pattern: track last departure, check clearance, schedule next. Station dispatch → junction merge timing. [Cross-domain]
6. **network.json is the only source of truth** — station names were stored only in entity attributes, invisible to Travel app. Build was overwriting them. Now Build preserves user data. Template folders are read-only. [Cross-domain]
7. **Template validation chain must have timestamps** — Save > Compute > Test. If model changed since Compute, reject. If recomputed but not retested, reject. All dt UTC. [Cross-domain]

## Scars

- The 500mm hallucination cost hours across multiple sessions. Each time it was fixed, compression destroyed the lesson. Now it's in memory, in code guards, in visual flags, and in the crew flag system. The cost was high; the infrastructure it produced is permanent.
- Entity name "corruption" during animation was a false alarm — SketchUp transparent operations hide names in the entity browser, but they're intact after stop. Hours spent investigating a non-bug.
- Build overwriting station_names was invisible until the Travel app needed them. The rule "Build must not destroy user data" seems obvious but wasn't enforced.
