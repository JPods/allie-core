## Lessons for Allie

1. **Ring connector length = 1000mm is a self-check** — In traffic_circle7, short ring
   connectors are exactly 1000mm when both endpoints are on the ring centerline. Length ≠ 1000mm
   (e.g., 1118mm = old value) means an endpoint is at a switch-box offset ~500mm inside the ring.
   Check connector length before declaring a ring-topology fix complete. Domain: [SketchUp]

2. **Lesson re-learned = missing documentation** — Ring junction endpoint rule was learned
   on JPods_station_parking (2026-06-08) and again on traffic_circle7 (2026-06-09). The second
   time paid for Axiom 21 and the formation-tracks-recovery.md guide. When a lesson is learned
   twice, the cost is writing the recovery guide so it cannot be learned a third time. Domain: [Cross-domain]

3. **Recovery guide structure that works**: symptom table → EP coordinate table → Bezier recipe
   table → Python script → git restore. Future sessions need the Bezier recipe first (which tangent
   vectors), then verification (gap check + length check). The Python script makes recomputation
   a 30-second task. Domain: [SketchUp]

## Scars

**Paid twice:** Ring junction off-ring endpoints. First time on JPods_station_parking (switch-box
offsets for EP6/EP7/EP11), second time on traffic_circle7 (same root cause, different template).
Both times the visual symptom was "ribbon crosses the guideway." The scar: extracted.json captures
model geometry that includes switch-box positions — these are NOT the ring centerline. Always
derive ring junction coordinates from the arc's own FIRST/LAST pts, not from extracted.json
approach-track endpoints.
