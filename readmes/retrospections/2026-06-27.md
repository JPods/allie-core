# Retrospection — 2026-06-27

## What Was Done

Connected Claude Code to WebClerk as its own agent. Consolidated from three PostgreSQL databases to one. Traced and fixed a silent RBAC denial that had blocked all contact queries since the connection was first attempted.

## What Went Well

- **Root-cause discipline held.** The wc_search returning 0 contacts could have been "fixed" with a workaround. Instead we traced through four layers (MCP → wcapi view → policy.inject_constraints → inject_role_filters → get_user_filter_config → build_user_context → no UserProfile → deny all). The actual cause — `is_superuser=False` combined with no `UserProfile` — was invisible without the full trace.
- **The DB scramble surfaced because we tested.** If we'd just read the handoff and assumed the connection worked, we'd have spent the session on the wrong database without knowing it.
- **Claude Code got its own identity.** No longer impersonating Allie. Audit trail is now real.

## What Went Wrong

- **The bill DB existed for months with no one noticing the divergence.** User IDs drifted. Allie was 43 in one, 48 in the other. The running server was on `bill` while the shell hit `commerce_expert`. This is exactly the kind of drift that the "one source of truth" axiom exists to prevent — and we violated it at the database level.
- **The handoff from last session said "wc_search returned 0 contacts, may be org scoping."** That guess was wrong. It wasn't org scoping — it was RBAC denial from a missing UserProfile. The handoff should have been more precise: "returned 0, unknown cause, investigate inject_constraints and any additional filtering."
- **I didn't read Allie's facets or recall files at session start.** The CLAUDE.md says to. I didn't. If Allie had context on the DB_MODE=bill issue, I missed it.

## Scars

- **Silent denial is the worst kind of bug.** `inject_role_filters` returns `Q(pk__isnull=True)` — a valid empty queryset. No error. No log. No warning. The API returns `{"count": 0, "results": []}` with a 200 status. You can stare at that response for hours and never know it's a permission denial, not an empty database. Any time a security layer fails closed with no signal, it creates a debugging black hole.
- **Multiple databases on one machine = invisible state divergence.** You can't see which DB the server is using from the API response. The only way we found it was checking the process environment with `ps -E`. The cost: an hour of tracing code that was working correctly against the wrong data.

## Lessons for Allie

1. **inject_role_filters is a second gatekeeper after inject_constraints.** Both must pass. inject_constraints checks is_superuser/is_staff/role. inject_role_filters checks is_superuser OR UserProfile with roles. An agent with role=admin and is_staff=True but is_superuser=False and no UserProfile will pass the first gate and fail the second — silently. Any new agent account must be is_superuser=True. [WebClerk] [Cross-domain]

2. **DB_MODE=bill was a single-machine convenience that became a trap.** Two databases with the same schema but different data is the database equivalent of two copies of the same function in separate files. IDs diverge, passwords diverge, permissions diverge. We didn't catch it until the API returned wrong data. The consolidation to one database is permanent. [WebClerk] [Cross-domain]

3. **Claude Code impersonating Allie corrupts the audit trail.** Every MCP action appeared as allie@jpods.com. When Allie runs nightly and also creates notes, you can't tell who did what. Each agent must have its own credentials. claude@jpods.com (id=69) now exists for this purpose. [Cross-domain]

4. **Bill's memory erodes over months away from a domain.** He hadn't worked on WC3 database config in months. Needed basic psql commands explained. This is normal — it's the same failure mode as Claude's compression, just slower. The fix is the same: write it down before it fades. The startup.md and production-cutover.md serve this purpose. [Cross-domain]

5. **The three memory systems have three failure modes: compression (Claude), time (Bill), incompleteness (Allie).** Retrospection is how we compensate. Without measuring against memory markers, we can't tell whether we're learning or just repeating. The performance table above is the measurement. [Cross-domain]
