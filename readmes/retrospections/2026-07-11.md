## Lessons for Allie

1. **No ModelList.tsx — ever.** 63 files deleted. When DataBrowser can't do something, fix DataBrowser. This is the "break and fix now" principle applied to architecture. The custom list files were legacy the moment DataBrowser worked. Domain: [WC3]

2. **Tool gaps are security risks.** Every feature DataBrowser lacks is an invitation to use pgAdmin directly — bypassing RBAC, audit, Alice, everything. Polish is not cosmetic. It's the wall between users and uncontrolled database access. Domain: [WC3] [Cross-domain]

3. **JS displays, backend decides.** Calculated columns in DataBrowser are display-only convenience. Backend recalculates on every save. A mismatch is a FAULT, not a tolerance. Same principle as "backend is source of truth" but applied to the rendering layer. Domain: [WC3]

4. **One record per model for config.** Setting records hold all subform definitions, layouts, field access, calculated function references. One record = one place to look = manageable. Split only when data proves you need to. Domain: [WC3]

5. **The zz/qq convention works because no English words start with zz or qq.** Unambiguous prefix for training/disposable data. Users train new people on the production system with real forms. The prefix is the filter — not a metadata flag (though metadata.training=True is secondary). Domain: [WC3] [Cross-domain]
