## What changed

### OQ-1 — Trip file staleness (jpod_followme_exporter.rb)
- Added `followme_generated_at` field to every trip file.
- `purge_stale_trip_files(model, current_generated_at)` now runs after every
  followme.json export. Stale or unsigned trip files are deleted immediately and
  the operator is alerted. No legacy support.
- `export_all_trip_jsons` reads the followme timestamp once and stamps all trips.
- New helper: `read_followme_generated_at(model)`.

### OQ-2 — Semantic internal connection naming (jpod_followme_exporter.rb)
- `build_v2_document` now generates internal connection keys as
  `"#{sid}_#{conn_name}"` (using the physical guideway's own `connection_id`).
- Old ordinal naming (`_internal_0`, `_internal_1`) removed. No legacy support.
- Stable across station template edits as long as the guideway connection_id
  does not change.

### OQ-3 — Direction strings (jpod_followme_exporter.rb)
- Guideway tracks: `"out"` (index 1, forward from→to) and `"in"` (index 0, return).
- Internal tracks: `"u_turn"`, `"out"`, or `"in"`.
- Unicode arrow strings (`"S097→S098"`) removed.

### OQ-4 — nora.json abbreviated log (jpod_followme_exporter.rb)
- New method: `append_nora_log(model, trip_data)`.
- New constant: `NORA_LOG_MAX_ENTRIES = 500`.
- New path helper: `nora_log_path(model)` → `<model_dir>/<model_name>.nora.json`.
- Schema: `jpods-nora-log-v1`. Each entry: trip_id, nora_id, exported_at,
  followme_generated_at, platform_start, platform_end, line_count, anomalies: [].
- `anomalies: []` is reserved for future Nora-to-fleet observation sharing.

### OC-5 — Production geometry strip (deferred)

- Logged in `readmes/issues.md` as issue #12.
