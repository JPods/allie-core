# Retrospection — 2026-07-21

## What Was Done
- IT15 hardware setup: Ubuntu 26.04 LTS installed on GEEKOM IT15 (Intel Core Ultra 9 285H)
- Hostname set to "andi" during install
- OpenSSH server enabled
- Multiple hardware/display issues diagnosed and resolved

## Scars Paid
1. **Monitor compatibility**: First monitor (LG, HDMI-only) couldn't display IT15 output — likely resolution/refresh rate mismatch. Wasted time troubleshooting what appeared to be a dead machine. Lesson: always try a second monitor before assuming hardware failure.
2. **dd command failure**: `sudo dd` can't run from Claude Code's Bash tool (no interactive password). Wasted a full boot cycle with an unflashed USB drive. Lesson: use balenaEtcher, not dd.
3. **BIOS key timing**: Tapping DEL/F7/F12 during boot is unreliable — the window is too short. The Windows Shift+Restart → Use a Device path is far more reliable. Lesson: when Windows is available, use it to boot from USB.
4. **Forgotten credentials**: Bill couldn't remember the username/password set during install. Lesson: write it down immediately, or use a standard like `bill` / known password.
5. **Ubuntu installer navigation**: Space toggles checkboxes, Tab moves between sections, Enter activates buttons. Not obvious — Bill couldn't select OpenSSH until we figured out Space.

## Lessons for Allie

1. **balenaEtcher is the proven path** — dd failed due to sudo requirements in automated environments. balenaEtcher gives visual progress and just works. Always recommend it first. Domain: [SYS]

2. **Monitor issues mask working hardware** — The IT15 was working the entire time. The first monitor couldn't display the output. Before diagnosing a "dead" machine, try a different monitor or HDMI port. Domain: [SYS]

3. **Manual install is the specification** — Every step Bill struggled with (ISO flashing, BIOS navigation, USB boot selection, installer UI, credential management) becomes a requirement for the future Andi one-touch installer. The pain is the spec. Domain: [Cross-domain]

4. **Andi is a product, not a service** — Bill clarified that Andi ships with hardware. Each business gets their own box + AI + data. This is Desktop Hosting realized. The opt-in sharing with Allie follows CarryOn sovereignty — patterns shared, raw data stays local. Domain: [Cross-domain]

5. **Ubuntu installer uses Space/Tab/Enter** — Space toggles checkboxes, Tab navigates sections, Enter activates buttons. Arrow keys select within a section. Document this for the installer tool. Domain: [SYS]
