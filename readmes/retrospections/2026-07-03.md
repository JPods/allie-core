## Lessons for Allie

1. **The `data` field name was a collision waiting to happen** ‚Äî When a model field has the same name as the API envelope wrapper, every layer of the stack needs special-case handling. The fix (rename to `config`) is mechanical but affects 80+ files. The lesson: reserve common words (`data`, `type`, `status`) for API/framework use, not model fields. [Cross-domain]

2. **Celery tasks without decorators are invisible** ‚Äî All functions in `tasks.py` existed and were scheduled in Beat, but none had `@shared_task`. They looked wired but never ran. Symptom: Pending records with `dt_processed=0` forever. The lesson: verify task registration, not just file presence. `celery inspect registered` would have caught this. [WC3]

3. **Chrome 150 broke the debug port pattern** ‚Äî Previous Chrome versions accepted `--remote-debugging-port` alone. Chrome 150 requires `--user-data-dir` too, and silently ignores the port flag without it. No error message. The lesson: when a tool stops working after an update, check the release notes for changed requirements. [SYS]

4. **Django only discovers models in `models.py` or `models/__init__.py`** ‚Äî `models_alice.py` was a separate file with 3 models. Django never saw them. The fix is a one-line import. The lesson: Django's model discovery is file-name-based, not directory-based. If you split models across files, you must import them. [WC3]

5. **Vector store size doesn't equal quality** ‚Äî We grew Alice from 0 to 4,656 chunks. But we haven't measured whether searches return the right content. Adding more content without measuring retrieval quality is activity masquerading as progress. The lesson: build the scoring test before the next indexing run. [Cross-domain]
