## Lessons for Allie

1. **python-decouple env var precedence** — env vars override .env files silently. A stale `DB_MODE=bill` in the shell caused all queries to hit the remote database. Fix: hardcode when there's only one valid option. [Cross-domain: any decouple-based config is vulnerable to this]

2. **GL posting must be user-initiated** — invoices/payments are editable until explicitly journalized. After journalizing, only reversals allowed. This is a fundamental accounting principle, not a technical choice. [Cross-domain: applies to any financial record in any WC deployment]

3. **Tax and shipping are fixed user-entered values** until external API providers are offered. No calculated tax on save. Adjustment factor for actual vs estimated comes later (was in wc2). [Cross-domain: simplifies initial deployment]

4. **Pending is the inventory mechanism** — not direct reservation. Line save → Pending with quantity buckets → Celery 30s → InventoryLayer. `reserve_inventory_for_order()` is a separate visibility aid only. [Cross-domain: all programs should understand this pipeline]

5. **refs and metadata are secondary to PKs** — denormalized caches, never authoritative. FK wins on conflict. Nightly Celery audit catches drift. [Cross-domain: fundamental wc3 architecture principle]

6. **Backend is source of truth** — React displays and submits, server validates and decides. If they disagree, server wins. [Cross-domain: all future UI work]

7. **ContactFactory was broken** — used `username`/`first_name`/`last_name` but Contact model uses `email`/`name_first`/`name_last`. Fixed. WarehouseFactory had `ida` but Warehouse has it as read-only property. Fixed. [Lesson: test factories must match current model schema]
