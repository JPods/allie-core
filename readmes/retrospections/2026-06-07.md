## Lessons for Allie

1. **`Geom::ORIGIN` is not accessible inside lambdas in SketchUp Ruby** — use
   `Geom::Point3d.new(0,0,0)`. The constant `Geom::ORIGIN` fails in lambda/block scope
   inside the JPods module. Every future use of the origin point inside a lambda must use
   the explicit form. Domain: [SketchUp]

2. **ene_railroad solid geometry: no standalone edges** — FollowMe-extruded solids have
   edges only via `face.outer_loop.edges`. Any indicator geometry that code must find
   (direction edge, reference edge) must be a standalone edge NOT welded to face geometry.
   Add this to station model authoring rules. Domain: [SketchUp]

3. **When SketchUp makes a component from geometry, local origin ≠ edge endpoint** —
   Do not require `vertex.distance(Geom::ORIGIN) < tol` for externally placed components.
   Use the component's world insertion point to determine the "from" end of the indicator edge.
   Domain: [SketchUp]

4. **Explicit datums must be visible to humans** — Axiom 10 requires explicit model datums
   over derived references. But a datum hidden inside solid geometry is not truly explicit —
   it's hidden from the designer who must maintain it. Placing the vector_in indicator as a
   visible component outside the track group but inside its envelope solves both the
   human-visibility problem and the code-accessibility problem simultaneously. Domain: [Cross-domain]

5. **Diagnostic condition must check the right entity attribute** — `_diag` was false for
   3 iterations because it checked `entity.definition.name` (the ene_railroad component name)
   rather than `entity.layer.name` (the JPods tag). When a diagnostic doesn't fire, check
   the condition first, not the search logic. Domain: [SketchUp]

---

6. **Sally platform→parking check must be BFS, not direct-successor** — station_line_end
   routes from `gw_platform` to parking via a multi-hop hold loop (8 tracks). A direct
   successor check will always flag this as broken even when it is correct. BFS through
   the lines.json successor graph is the right check for reachability.
   Domain: [SketchUp]

7. **Pass-through topologies have no platform/parking — and that is correct** —
   traffic_circle7 is a pure routing station; pods never park. Sally's parking requirement
   must check for topology type before flagging. Marker: `pass_chains` present AND
   `hold_loop_chain` absent in lines.json. Generalizes to any future routing-only template.
   Domain: [SketchUp]

8. **Proof checks should read from extracted.json, not re-scan geometry** — Proof had its
   own `_find_direction_vector` call for the `gw_cp_in_*` vector_in check, independent of
   what `populate_from_open_template` had already determined. Any check Proof makes about
   a property that was computed during extraction should read `extracted.json['that_field']`,
   not re-derive it. Avoids drift between extraction result and proof verdict.
   Domain: [SketchUp]
