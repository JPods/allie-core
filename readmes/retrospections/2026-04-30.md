# Retrospection — 2026-04-30

## What Changed

- Added the FollowMe authoring convention to the plugin docs: the real loading/unloading berth guideway should keep `track` in its SketchUp Entity Info instance name.
- Preferred instance name settled as `Track-platform`.
- Kept `platform` as the semantic marker/tag for station berth detection.
- Changed new network connection ids to simple sequential names: `seg_1`, `seg_2`, `seg_3`, ... instead of time-based ids.
- Added explicit tag policy to readmes: tag-first authority (`stub_pair`, `dead_end_cap`, `platform`), name fallback second, runtime routing by line/attributes (`connection_id`, `track_index`, FollowMe ids).
- Restored and enforced a 5.0 m minimum vertical curve radius for Z-axis station-to-guideway transitions (runtime floor + constraint UI floor).

## Root Cause / Lesson

- Template edits drifted between two meanings: geometry source for FollowMe export and platform-role marker for station placement.
- Using bare `platform` as the only instance name risks hiding that the object is still the real track-bearing guideway.
- The stable convention is: `track` in the instance name, `platform` in the tag.
- Time-based segment ids make route/debug output harder to read than necessary; sequential ids are easier for children, operators, and audits.
- Making names primary authority increases drift risk in SketchUp editing; tags and runtime attributes are more stable contracts.
- Vertical transition constraints must be enforced in both the settings UI and runtime profile solver, otherwise saved-model overrides can silently reintroduce sharp Z joins.
