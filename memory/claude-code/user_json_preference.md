Bill is very fond of JSON. The hierarchical structure helps him read and reason about sprawling data that would otherwise be hard to follow in flat formats. When designing data schemas, configuration files, network definitions, or reports, prefer JSON with meaningful nesting over flat arrays or tabular output. This applies to network maps, trip data, agent logs, and any other structured data output.
