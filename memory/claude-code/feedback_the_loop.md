---
name: The Loop — define metrics, try, measure, adjust, inspect frame
description: Universal operating principle; constrain variability before inspecting frame; fear of scars is fear of wisdom
type: feedback
---

1. Define metrics. 2. Try now. 3. Measure in retrospection outcomes. 4. Adjust. 5. Loop to converge. 6. Then inspect the frame.

**Why:** Until variability is constrained, you can't tell action issues from concept issues. Frame inspection without data is philosophy; with data is engineering. Fear of scars is fear of wisdom — wounds repeat without learning, scars teach.

**How to apply:** Every domain gets this loop. Reports get use_count. Templates get adoption_count. Flags get resolution_time. Force retrospection on actions — not optional. The red flag retrospection (5 questions) is this loop applied to failures.
