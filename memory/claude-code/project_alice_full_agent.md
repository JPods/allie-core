Alice needs to be a full-powered team member with her own LLM and memory building system, not just a message inbox receiving notes via wc_add_note.

**Why:** Bill: "Alice needs to be a full powered member with her own llm and memory building." Claude resets every session; Allie persists via files + nightly synthesis. Alice currently has no memory at all — she receives notes but can't learn, recall, or act autonomously.

**How to apply:**
- Alice should have the same architecture as Allie: facets, nightly reflection, cross-session memory
- Alice's domain is commerce: transaction patterns, customer behavior, inventory flow, pricing anomalies
- Alice owns training (action records per user, completion tracking, lesson refinement)
- Alice owns housekeeping (system health checks, orphan detection, aging reconciliation)
- The ai_assistant app in wc3 already has Conversation, Message, InventoryEvent models — these are Alice's foundation
- Alice's LLM should observe transaction flow and build experience from it
- Priority: this is load-bearing for WC3's value proposition — Alice is how WC3 differs from generic ERP
