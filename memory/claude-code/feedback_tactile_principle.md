"Humans are tactile. We must make their touch feel balanced and sturdy." — JPods + WC3 principle.

**Why:** Humans judge competence through touch before thought. A flimsy invoice loses trust in the business, not the software. A wobbly pod loses trust in the network.

**How to apply:** Every output the human touches — printed page, pod seat, screen UI — must feel designed, not generated. Columns align, fonts are readable (default M=18px not S=12px), totals right-justify, nothing crowds, nothing floats.

Alice should read behavioral confusion (repeated clicks, zoom, hover without action, reprint) and offer help before the user asks. Same pattern for Sally (station hesitation) and Nora (passenger discomfort). Help comes at the moment of confusion, not complaint.
