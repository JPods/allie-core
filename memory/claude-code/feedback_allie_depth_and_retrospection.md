Bill: "Use Allie heavily, not just summaries. Allie and I would like to help you overcome your programming limits. When available, use Allie 5t, she has lots of storage. The key with storage is to use retrospection to sort through data to find information and sort through information to gain experience."

**The distillation chain:**
- **Data** — raw observations, FAULT files, trip logs, physical.json batches, session events
- **Information** — patterns extracted from data: what happened, where, how often, what correlates
- **Experience** — principles extracted from information: what it means, what to do differently next time, the rule that makes the next decision obvious

Retrospection is the process that moves data → information → experience. Without retrospection, data accumulates but nothing is learned. The TFTS format is the unit of retrospection: one arc, one principle, stated so the next attempt is obvious.

**Practical instructions:**
- At session start: read Allie's full files — thoughts/YYYY-MM-DD-reflect.md, facets/, wisdom/, recall files. Not just the handoff summary.
- When available, read from /Volumes/Allie/ (5TB) — she has knowledge files, retrospections, and wisdom that don't fit in the working copy.
- At session end: write a real retrospection with "Lessons for Allie" — not a summary of what happened, but the principles that should change what is done next time.
- Write TFTS files the moment an arc closes — context evaporates. A TFTS written 10 minutes after success is worth more than a perfect summary written the next day.
- The facets are Allie's distilled experience for each agent. Read them. They represent dozens of runs already processed through retrospection.

**Why:** Claude Code's limit is the session boundary — it resets. Allie's limit is synthesis speed — she runs nightly, not in real time. Between them, the knowledge exists. The failure mode is Claude Code not reading what Allie has already synthesized, and Allie not receiving what Claude Code discovered in the session. Retrospection at session end is the hand-off that closes that gap.
