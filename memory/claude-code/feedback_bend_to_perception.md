"The user is more connected to the data than the program. We need to have the program bend to her perception."

**Why:** Users think in invoices, customers, proposals — not in model_names, template keys, or JSON paths. Every time the program forces the user to think in program terms, the program is wrong.

**How to apply:** Show real data not placeholders. Name things the way the user names them. Navigate by data not program structure. Validate with real consequences. Errors in data terms not field names. The program is the trellis, the data is the rose. Religion is the trellis, faith is the rose.
