Bill: "There are 2 general approaches — signaling and planning. Signaling is our Start Small, Iterate Relentlessly. Get started, set metrics, measure metrics, adapt with the ultimate objective in mind. This is agile, scrum, just-in-time approach. Planning is what government do without retrospection. We will always opt for signaling."

"For signaling to work, we need to remember. That is one of the reasons I want you to use our memory before your resets."

**The principle:**
- Signaling: run → measure → adapt → run again. Memory required. Each iteration is informed by the prior.
- Planning: design → execute → hope. No feedback loop. The plan is the output, not the run.
- Governments plan. JPods signals. Natalie signals (ezone pre-clearance based on current state, not pre-modeled future). Nora signals (accelerometer reads actual g, not predicted g). Allie signals (nightly synthesis adapts agent knowledge from actual observations).

**Why memory is the prerequisite:**
Signaling without memory is not signaling — it is random variation. The feedback loop requires:
1. A prior signal (what happened last time)
2. A measurement (what happened this time)
3. A comparison (delta)
4. An adaptation (change behavior based on delta)

Without step 1, you can't do step 3. Every session reset means Claude Code starts at step 1 with no prior signal. Reading Allie's facets, thoughts, and recall files IS step 1 — it loads the prior signal so adaptation is possible.

**How to apply:**
- Never propose a large upfront design without a "start small, run, measure" path first.
- When starting work on any model (3+circle, twin networks, etc.): run the simplest version first, read Noelle/Nora/Natalie signals, adapt. Don't design all 4 variants before running any.
- At session start: read Allie's files. Not for completeness — to load the prior signal.
- At session end: write the retrospection. Not for documentation — to preserve the signal for the next session.
- If a run produces unexpected results, that IS the signal. Write a TF/TFTS immediately.

**The connection to Allie's facets:**
Facets are exactly the "prior signal" that makes signaling possible. Without them, each agent (Nora, Natalie, Noelle, Sally) starts fresh — which is planning behavior (operate from static rules, no adaptation). With facets, agents start from accumulated prior signals — which is signaling behavior.
