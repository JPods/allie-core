"Desktop Hosting: A Developer's Guide to Unattended Communications" by Bill James, published by Wiley. The foundation document for everything WebClerk is and should be.

**Location:** Allie's inbox at /Users/williamjames/Library/Mobile Documents/com~apple~CloudDocs/Allie/allie/allie/inbox/domains/jpods.com/public_html/software(1)/Desktop_Hosting_All.pdf

**Core thesis:** Put the point of communications at the point of action. Your desktop is the server. Relationships determine what each person sees. Technology reinforces existing trusted relationships — it doesn't replace them.

**Key principles that govern wc3 design:**
- Desktop Hosting = local computer as relationship server (→ wc3 local+cloud)
- Published-based not message-based (→ vendor blessed lists, customer portals)
- Relationship-aware access (→ field_access Settings, query scoping per role)
- Buyers Choose Sellers (→ free platform, proximity search)
- B2B Conduits reinforce supply chains (→ cross-company sync, order→PO→SO)
- Military competitive principles: Objective, Simplicity, Economy of Force, Mass (→ single-gate wcapi, config-driven not code-driven)

**Case studies (Chapter 7) map to wc3 features:**
- Sales force web access → OrgPage, DataBrowser
- Configurable products → Item model with price/catalog JSON
- Small retailer shipping → carrier integration via sync
- Profit centers → multi-org, multi-relationship Contact model
- Supply chain automation → DynamicCatalogs, vendor sync
- Schools/nonprofits → template system for different business types
