# Session Log — 2026-05-06

## Goal

Fix three Route-Time GUI bugs: vehicles appearing at traffic circles during simulation load animation, lines not redrawing after dragging structures in loaded networks, and Alt+drag panning the map instead of moving structures.

## Accomplished

- **`_line_coords` stale coordinate bug** (`api.py:_line_coords`, lines 442–448)
  - Root cause: every line loaded from a `.jpd` file has `coordinates` set (even a 2-point list), making the guard `if line.coordinates:` always truthy — so `_line_coords` returned frozen file positions even after `rotate_station` moved the nodes.
  - Fix: only use stored coordinates for interior waypoints (`len(coords) > 2`); always derive endpoints from live `start_node`/`end_node` positions.
  - Same fix applied to `io/jpd_writer.py` so saved files also reflect moved positions.

- **Loading animation vehicles on wrong guideways** (`simulator.js:_startLoadingAnim`)
  - Root cause: animation placed pods on ALL lines including internal station lines and traffic circles.
  - Fix: filter to `!f.properties.structure_id` — only inter-station connector lines get animated pods.

- **Alt+drag panning the map** (`app.js`)
  - Root cause: `map.dragging.disable()` was missing from the single-structure Alt+drag branch (Branch A). Also, when Alt was held but cursor wasn't over a structure, the handler returned early without `preventDefault`, so Leaflet still panned.
  - Fix: disable map dragging on `keydown Alt`; re-enable on `keyup Alt` (if no move active) and on `mouseup` (if Alt not still held). This makes Alt a global "no-pan" mode key.

- **harvest.py updated** to read `sessions/YYYY-MM-DD.md` and include Accomplished/Next/Open Questions as the first section of the harvest — debugging context now flows into Allie's nightly reflection.
