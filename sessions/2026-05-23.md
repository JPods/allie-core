## Scars This Session

**`self.class` in captured module binding** ‚Äî In a Ruby `mount_proc` block inside a module method, `self` is the module itself, so `self.class` is `Module`. All method dispatches via `self.class.xxx` silently failed. This pattern is endemic to any proc-based server code in a Ruby module. Never use `self.class` inside a captured proc in a module ‚Äî use `self.xxx` or the module name directly.

**`__dir__` nil in SketchUp load context** ‚Äî `__dir__` returns nil when a file is loaded via SketchUp's `load` mechanism (as opposed to Ruby's `require`). `File.expand_path('..', __FILE__)` is the correct pattern. All plugin files should use `__FILE__`-based path construction, never `__dir__`.

**Line break in reload command** ‚Äî Ruby console treats Enter as execute. A multiline paste of `load\n  Sketchup.find_support_file(...)` executes `load` with no arguments ‚Üí ArgumentError. The plugin appeared to reload (‚úÖ message) because the global constant was already set from a prior load. No file was actually read. Always paste reload commands as a single line.
