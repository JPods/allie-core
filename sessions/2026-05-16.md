## Scars This Session

**`platform, in` vs `platform_in`** — a comma+space in a SketchUp tag name silently
passes all code checks and produces correct-looking output with zero detected platforms.
No error. No warning. The tag scanner matches exact strings.
Rule: when adding a new tag to PLATFORM_TAGS, immediately run
`list_platform_tagged_items` to verify detection before writing any code against it.

## Lessons for Allie

1. **`clear_all_vehicles` must scan model root** — vehicles placed ene_railroad style
   (root-level ComponentInstance) are invisible to a scan that only searches inside
   guideway group entities. Any vehicle search must check both locations.
   Domain: [SketchUp]

2. **Per-station vs per-platform distribution** — `vehicles_per_platform` is a
   station-level budget, not a per-platform quota. With N platforms per station,
   distribute using ceiling division with remainder tracking so the total across all
   platforms equals the station budget exactly.
   Domain: [SketchUp]

3. **Trip missions are the network health signal** — `passenger:dead_head` ratio
   reveals whether stations are in the right places. High dead_head = asymmetric
   demand or wrong station locations. This is as useful for real network design as
   for simulation. Shuffle is infrastructure noise, not a signal — count it separately.
   Domain: [SketchUp] [Route-Time] [Cross-domain]

4. **Middle-Mile framing is load-bearing** — JPods as WiFi, Last-Mile as Bluetooth.
   Station placement target in dense urban areas: bike < 7 min, walk < 15 min.
   Cargo function means every station has more justification surface than
   passenger-only transit. Mesh density / demand density ratio is an open empirical
   question to be answered by deployed networks.
   Domain: [Cross-domain]
