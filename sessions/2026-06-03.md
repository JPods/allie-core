## Accomplished

- **Sally nil/[] semantics locked** — `hl_to_platform=nil` means legacy/derived (fall back
  to landing_chains); `[]` means intentional direct-park. Changed sentinel from `[]` to `nil`
  for legacy so the two cases are unambiguous. `on_maneuver_complete` checks `if landing.nil?`

- **Intersection-based landing routing implemented** — `_final_approach_tracks(hl_loop, lc)`
  finds the first track shared between the outer ring and landing_chain. Dispatches
  `partial_loop[0..exit_idx] + landing_chain_tail`. Sally now has ONE algorithm for all
  templates. Priority order: direct-park → intersection → to_platform fallback → legacy nil.

- **station_parking/lines.json: landing_chains added** — `in_cp0: {tracks: ["gw_cp_in_lead_0",
  "gw_platform_in1", "gw_platform_in2"]}`. Intersection at gw_cp_in_lead_0 (index 6 in
  hl_loop). Final approach = 7 partial loop + 2 platform approach = 9 tracks.

- **Platform departure fixed: no backward travel** — `_pts_tail_from_near(pts, near_pt)` clips
  a polyline at the closest projection of the pod's position, returns the forward tail only.
  Added to RubyNatalie. `build_maneuver_from_tracks` gained `clip_start: true` option.
  Hold_loop departure now passes `clip_start: true` — pod at high slot exits forward directly,
  never reverses to slot 1.

## Scars This Session

**Module placement** — `_pts_tail_from_near` was added to `JPodVehicleAnim` but called from
`RubyNatalie` as `JPodGuideway._pts_tail_from_near`. Both the module and the namespace were
wrong. Cost: one SketchUp reload cycle and a crash. Rule: helpers live in the module where
they are called. Check the enclosing `module`/`class` boundary before adding a method, not
just the surrounding context.

**LIFT_RE in method body** — Ruby raises SyntaxError for constant assignment inside a def.
This is a well-known restriction but easy to miss when adding a regex literal. Cost: plugin
failed to load entirely. Rule: regex patterns used inside a single method are local variables
(`lift_re`), not constants.
