### traffic_circle7 Show Formation Tracks fixed

**Problem 1 — short connectors crossing the guideway:**
gw_c_0_1, gw_c_1_2, gw_c_2_3, gw_c_3_0 had endpoints ~500mm inside the ring radius.
Root cause: extracted.json captured switch-box offset coordinates, not the ring arc
centerline. The ring arcs' own FIRST/LAST pts are the authoritative junction coordinates.
Fix: set each connector's endpoints exactly to the adjacent arc's FIRST or LAST pt.
Result: all 8 junctions at 0.0mm gap; all 4 connectors at exactly 1000mm length.

**Problem 2 — gw_in/gw_out straight diagonals:**
8 approach tracks were 2-pt straight lines. Fixed to 13-pt Bezier curves (h=chord/3).
sv = CCW ring tangent at diverge EP; ev = REVERSE of CP track direction.
All ring-side endpoints use the same EP coordinates as the connectors (0mm gaps).

## Lessons for Allie

1. **Ring junction endpoints come from arc FIRST/LAST, not switch-box geometry** —
   When short connectors in a ring template look wrong (crossing the guideway, length ≠ 1000mm),
   the endpoints are at switch-box offset positions inside the ring radius. Fix: set them to
   the adjacent arc's FIRST or LAST pt. Correct short-connector length = 1000mm is a built-in
   self-check. Domain: [SketchUp]

2. **Save_geometry() protection: >2 pts OR authored=true** — Any track in geometry.json
   with more than 2 pts (Bezier/arc) or with `"authored": true` is preserved when Generate
   Geometry JSON runs. Only length_mm is updated. This is the silent guardian — it works
   automatically without user action. Domain: [SketchUp]

3. **Four-layer protection is necessary because students will run Generate Geometry JSON** —
   The Console gate (Layer 1) stops most accidents. save_geometry() (Layer 2 embedded in code)
   stops the rest. notes.md and CLAUDE.md are for recovery when both fail. The order of cost:
   Console gate = free; save_geometry = silent; notes.md = one session; manual recompute = hours.
   Always build the cheap guards first. Domain: [SketchUp]

## Session 2 (continuation) — 2026-06-09

**Focus:** Template model animation — Sally test without network

### Problem addressed

Template models (.../track_formations/<fid>/model.skp) have:
- No platforms in followme.json (wrong schema)
- Empty path.json and map.json
- No BFS routing topology

`cmd_sally_standard_test` raised "No platforms — run Build first" on template models.
The goal: isolate station behavior testing to individual templates so defects can be
found before variables compound across a full network.

### Solution

Two-method approach: the animation engine needed a new entry point that bypasses the
path.json/map.json pipeline entirely.

**`start_for_template(model, template_lookup)` in jpod_vehicle_anim.rb:**
- Accepts a pre-built lookup (geometry.json → pts in world coords, keyed by station_id.track_name)
- Calls existing `build_fleet` — Sally hold loop path works without map.json
- Sets `@@template_mode = true` to trigger two-phase vehicle exit in dwelling loop
- Two-phase exit: (1) land at platform → dispatch originating chain; (2) at CP stub → erase
- `stop()` clears `@@template_mode = false`

**`cmd_sally_standard_test` template branch in jpod_console.rb:**
- Detects template model by path containing 'track_formations'
- Reads geometry.json → builds template_lookup (station_id.track_name keys)
- Places V1 directly via `model.entities.add_instance` (no guideway group needed)
- Sets `template_formation_id` attribute so dwelling loop can find lines.json
- Calls `start_for_template` instead of `start_animation`
- Poll watches for V1 erasure; `raise 'template_path_done'` exits network flow cleanly

### Key principle

Each animation chain test exercises one layer of the network:
- `parking_chain` — vehicle placement at correct slot
- `hold_loop_chain` — 3 circuits on outer ring
- `landing_chains` — promotion from loop to platform
- `originating_chains` — CP departure

Template animation makes these independently verifiable before network topology
multiplies the failure modes.

## Lessons for Allie

1. **Sally chain dispatch already worked in the tick** — `on_maneuver_complete` and
   `build_maneuver_from_tracks` already operated independently of map.json. The only
   missing piece was the lookup for template track pts. Once geometry.json is loaded
   into `@@lookup_cache`, the entire hold loop + landing + exit machinery runs
   unchanged. Domain: [SketchUp]

2. **Two-phase exit is the right pattern for CP departure** — "exit via CP" in template
   mode means: (1) land at platform (Sally parks pod), (2) dispatch originating chain
   (pod drives to CP stub), (3) erase entity. Three distinct states, each with a
   clear trigger. Don't collapse them. Domain: [SketchUp]

3. **`raise 'template_path_done'` as flow exit** — When two execution branches share
   a `rescue => ex` block, a descriptive exception string is cleaner than nested
   conditionals. Check `ex.message == 'template_path_done'` in rescue and treat it
   as success. Domain: [SketchUp]
