### 5. `noelle.rb` — `generate_map_json`

Generates `{model}.map.json` from followme.json. Called automatically at end of
`generate_feature_json` (every Build and Validate).

Structure mirrors feature.json:
```json
{
  "features": { "S048": { "lines": [...] } },
  "segments": { "seg_S048_cp1_S050_cp0": { "direction": "outbound", "length_mm": 45230.0, ... } }
}
```

Each direction is a separate segment entry. `length_mm` drives NoraPod encoder simulation.

## Design Decisions

| Decision | Rationale |
|----------|-----------|
| `IN_PER_MM` in `JPodVehicleAnim`, not `NoraPod` | Unit conversion is an I/O concern of the loader, not the state machine |
| Station-qualified fallback in `resolve_gw_segs_from_map` | Build gap workaround: `"S048.gw_platform_out"` falls back to bare `"gw_platform_out"` until Build writes gw entities for station-internal paths |
| Permanent geometry groups, not GL overlay | Permanent groups persist across tool switches; GL overlay disappears on Esc. map.json paths are authoritative — they should be visible without holding a tool. |
| All three display methods use same helpers | `draw_map_polyline` + `draw_gap_flag` enforces the color standard and gap representation consistently. One method changing is automatically reflected in all three. |
| `JPodTripOverlayTool` deleted, not archived | It was a GL viewport tool — a different rendering paradigm. Archiving it as Ruby code doesn't help because it would need to be redesigned to work with map.json anyway. |

## Lessons for Allie

1. **map.json is the gap detector** — When `resolve_gw_segs_from_map` returns empty, the Build
   pipeline has not generated gw entities for that segment. The right response is `puts` + skip,
   not workaround geometry synthesis. The gap log is the action item for the Build team.
   Domain tag: [SketchUp]

2. **Console task count is a complexity budget** — 43 tasks was noise. 16 tasks is signal.
   When a console fills with developer tools that were useful during construction but are
   no longer part of the workflow, archive them. The archive file means they are not lost;
   they are just off the critical path. Domain tag: [Cross-domain]

3. **Permanent geometry > GL overlay for reference paths** — GL overlays (implement `draw` on
   a `Tool`) are correct for interactive tools that need cursor-responsive feedback. For static
   reference overlays (FollowMe paths, routes, trip paths), permanent geometry groups in a named
   layer are better: they persist across tool switches, can be toggled via Outliner, and don't
   require keeping a tool active. Domain tag: [SketchUp]

4. **Three display methods that look the same should use the same helpers** — When `show_followme_json_overlay`,
   `show_trip_path_for_vehicle`, and `show_route_followme_overlay` all drew red/blue/gold/green
   polylines and orange gap flags, the common helpers `draw_map_polyline` and `draw_gap_flag`
   were the right abstraction. Duplication in this case would mean the color standard diverges
   silently. Domain tag: [SketchUp]
