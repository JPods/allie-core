# Session Log — 2026-05-17

## Goal

Eliminate three independent trip-building code paths and establish TripPlanner as the
single source of truth for all platform-to-platform trips. Create noelle_features.json
architecture so station behaviors are declared once and read everywhere.

## Accomplished

- **Identified three duplicate trip-building paths** in jpod_platform.rb,
  jpod_natalie_bridge.rb, and jpod_animator.rb — all computing routes independently
  from the same inputs, producing divergent results when routing rules changed.

- **Created noelle_features.json** — plugin-folder source of truth for station template
  behaviors (originating/landing/pass_far/pass_near per CP). Templates: station_thru_dip,
  station_line_end, station_solar.

- **Created feature.json generation in Noelle** (`noelle.rb: generate_feature_json`):
  reads plugin noelle_features.json, resolves per-station behaviors using `formation_id`
  JPods attribute as primary key (not SketchUp definition name — those are unstable).
  Writes `{model}.feature.json`. Wired into Build and Review Network.

- **Rewrote TripPlanner** to read feature.json instead of hardcoded Ruby constants.
  Added `generate_from_path` (bypasses empty model.path), `lookup`/`lookup_one_way`/
  `lookup_round_trip`, staleness check (regenerates if followme.json or feature.json
  newer than trip.json).

- **Fixed inter_station_connections** — was reading `followme['connections']` (empty
  top-level array). Actual data is at `followme['network_definition']['connections']`.
  Field names: `structure_id`/`stub` (not `structure`/`cp`). No status field.

- **Fixed adjacency** — was synthesizing reverse edges with the wrong connection ID.
  followme.json has a separate entry per direction. Fixed to one-directional only.

- **Fixed station_line_end routes** — arrival incorrectly used `gw_far_out` as platform
  destination. Corrected to `gw_stub_pair_0_in → gw_platform_in → gw_platform_parking
  → gw_platform`. Departure corrected to include `gw_uturn_1 → gw_far_main → gw_far_out`.

- **Fixed formation_id key lookup** — initial noelle.rb used SketchUp definition name
  (`model`, `model#1`, `model#2`, `model#3`) as template key. These are not stable.
  Fixed to use `formation_id` JPods attribute as primary key.

- **Delegated all callers to TripPlanner**:
  - `jpod_platform.rb`: `build_platform_round_trip` + `build_platform_one_way_trip`
  - `jpod_natalie_bridge.rb`: `build_station_loop_trip`
  - `jpod_animator.rb`: `show_route_followme_overlay`

- **Verified 6 trips correct** against feature.json routing tables:
  S049→S048, S048→S050, S051→S049, S051→S048, S049→S050, S051→S050.
  All segment sequences verified manually.

- **Updated CLAUDE.md** — added Axiom 8: feature.json applies to all environments
  (SketchUp, Physical, Route-Time).

- **Updated noelle.md** — added U-SK-006 (gw_far_out is departure track, not platform),
  U-SK-007 (one source of truth principle), U-ALL-001 (feature.json cross-domain).

- **Wrote retrospection** — 2026-05-17.md with lessons, scars, For Noelle section.
