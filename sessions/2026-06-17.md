## Continued This Session (second context)

### traffic_circle7 Transit Test — three bugs fixed

**Bug 1 — Run button inert:**
`JSON.stringify(formation)` in `renderModelTests` onclick embedded double quotes inside a
double-quoted HTML attribute, breaking the onclick handler. Fixed with single-quoted JS
string literals in console.html.

**Bug 2 — Vehicles moving CW instead of CCW:**
pass_chains list track names in CCW order but `pts_mm` for some arcs (e.g. `gw_c_1_1`,
`gw_c_2_2`) are stored in the reverse direction. Transit intercept was blindly concatenating
pts in stored order. Fixed in build_fleet transit intercept: before stitching each track,
compare `last.distance(pts.first)` vs `last.distance(pts.last)` and reverse if needed.

**Bug 3 — Vehicles not following Show Track lines:**
`show_track_overlay` searches for the formation as `ComponentInstance OR Group` with a
Pass 2 gw_*-child fallback. Console's `template_lookup` builder only searched
`ComponentInstance` with no fallback. If the formation is a Group, transforms diverge.
Fixed: console builder now mirrors show_track_overlay's two-pass search exactly.

**model.network.json guard:**
`generate_network_json` now returns early for template models (`model.path.include?
('track_formations')`). Template models only have lines.json + lines.computed.json.

**Result:** Bill confirmed: "Looks great." All 4 vehicles exit cleanly CCW. Transit Test PASS.

must check which end connects before appending. First surfaced in transit intercept.

search functions that should return the same transform diverged silently. Rule: any code
that needs the formation instance transform must use the same two-pass search pattern.
