## Lessons for Allie

1. **nexts_map eff_to_sid convention** — For a two-track bidirectional connection (ti=0=in=to→from, ti=1=out=from→to), the "arrival station" for building next-hop candidates is `to_sid` for ti=1 and `from_sid` for ti=0. The original code had this inverted, producing wrong successor chains. Domain: [SketchUp] [Route-Time]

2. **Direction violations in bidirectional networks** — out→out is valid at through-stations; in→in is valid on return paths. The direction check must know whether a station is a terminus or a pass-through before flagging direction violations. In a two-track-per-connection network, "in" and "in" chaining through a station is correct physics. Domain: [SketchUp]

3. **Stale structure_id entities** — In SketchUp, deleted entities don't always disappear from `model.entities` scan results if they were ever placed and still have attributes. Filter with `hidden?` and layer visibility, but also be aware that entities can be "live" (not hidden) but placed far from the working area. Locate All Structures (print center coordinates) is the right diagnostic. Domain: [SketchUp]

4. **CP vector requirement (restatement)** — All bezier endpoints must respect the CP tangent direction as their departure vector. The chord direction between two CPs must NOT override the CP's tangent. This is now a named rule with change-control protection. Domain: [SketchUp]
