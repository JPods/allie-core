principle: THE RULE FOR TRAFFIC CIRCLE CCW ENFORCEMENT:

  Layer 1 — Routing (map.json successors):
    lines.json pass_chains declares the CCW successor graph. generate_map_json MUST
    read it as v4 schema. Routing sends the vehicle to the CCW ENTRY of each arc.
    Without correct routing, enforce_ccw_ring_arcs! and proximity flip fight each other.

  Layer 2 — Lookup direction (enforce_ccw_ring_arcs!):
    Ensures arcs in the lookup are CCW before any maneuver is built. Safety net against
    beam_path or synthesis producing CW pts. Cross-product at pts[0]→pts[1] vs center.

  Layer 3 — Proximity flip (build_maneuvers):
    Orients each arc segment so pts[0] connects to prev_end_pt. With correct CCW routing
    AND correct CCW lookup, dist_fwd≈0 for every ring arc → no flip needed → stays CCW.
    The flip is a connectivity aid, not a direction enforcer. When Layers 1+2 are correct,
    Layer 3 never flips ring arcs.

  The three layers must be consistent. If routing (Layer 1) is CW, Layer 3 will re-reverse
  every CCW lookup arc (Layer 2) back to CW. enforce_ccw_ring_arcs! cannot fix a CW
  routing graph — it only fixes the stored direction, not the traversal order.
