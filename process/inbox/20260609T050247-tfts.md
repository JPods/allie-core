principle: Two rules, both must hold for every Bezier in this codebase:

  1. COORDINATE FRAME — geometry.json pts_mm are formation-LOCAL (station_xf_inv
     applied at extraction). For template-model display: eff_xf = inner_xf only
     (world_xf = identity). For network-model export: eff_xf = world_xf * inner_xf.
     Flat templates have no inner_xf → use identity in both contexts.

  2. BEZIER HANDLE CONVENTION (ene_railroad / Axiom 11) —
       B1 = p0 + h*sv   where sv = vehicle travel direction at p0 (departure)
       B2 = p3 + h*ev   where ev = REVERSE of vehicle travel direction at p3
     The Bezier arrives at p3 FROM B2, so ev = -arrival_direction.
     Using ev = arrival_direction produces a 180° loop. Always verify ev is
     pointing BACK toward the FROM end, not forward toward the TO end.

apply_to_all_models:

  - Storage direction may differ from vehicle travel direction. Derive sv/ev in
    STORAGE direction: sv = direction from FIRST toward LAST if vehicle travels

  - Handle scale h = chord/3 gives smooth transitions. Adjust only if the curve
    bows too far (use h = chord/4 or chord/5 for tighter transitions).
