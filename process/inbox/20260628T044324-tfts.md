principle: The working waypoint Z pipeline (2331a6c) has five stages. Each stage has ONE job.
           Adding anything after the smoother breaks the pipeline.

           Stage 1 — DESIRED Z (piecewise linear)
             Piecewise linear interpolation through [CP_start, WP1, WP2, ..., CP_end].
             Each waypoint's Z comes from the beam_z entity attribute (terrain + CLEARANCE_HEIGHT
             + BEAM_DEPTH to match the beam-top anchor frame). Knots are placed at the
             closest path point to each waypoint XY. This is the TARGET profile.

           Stage 2 — GRADE CLAMP
             Grade envelopes run between adjacent anchor pairs (CP→WP, WP→WP, WP→CP), not
             between distant CPs. Each span gets its own grade_fwd/grade_bwd/ceil_fwd/ceil_bwd.
             min_z_bounds = max(floor_z, grade_fwd, grade_bwd). No hard_floor_z — raw terrain
             mesh is noise, not signal. Smooth guideways are primary; columns absorb terrain.

           Stage 3 — GAUSSIAN SMOOTH
             smooth_profile_z with curve_radius = MIN_Z_CHANGE_DIAMETER (default 60m, user
             adjustable via console Build Profile controls). Smooths the clamped profile.
             NO RE-CLAMP afterward. The smoother's output IS the answer at waypoint knots.
             Re-clamping undoes the smoothing at exactly the points that need it most.

           Stage 4 — ENDPOINT BLEND
             Hermite smoothstep blends the first/last 10m (or 35% of first/last anchor span)
             toward the CP anchor Z. Ensures the beam meets the station stub smoothly.
             Only affects the first and last spans — never reaches interior waypoints.

           Stage 5 — NOTHING
             No pin. No arc. No post-smoother waypoint correction. Every attempt to
             modify beam_z after the smoother (pin, parabolic arc, cubic Hermite blend)
             creates worse defects than the smoother alone. The smoother IS the
             waypoint transition mechanism. Trust it.
