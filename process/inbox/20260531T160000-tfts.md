# TFTS — 2026-05-31T16:00:00Z

problem:   Division of authority for lines.json and map.json/path.json was implicit —
           no enforcement in code and no clear documentation. eps[] was being overwritten
           by auto-scan tools; Noelle used unsigned templates without objection.

arc:
  - try:      use "note" field in lines.json to describe authorship intent
    result:   worked as documentation but not enforced — tools ignored it
    revealed: without machine-readable authorship record, any tool can overwrite eps[]

  - try:      add eps_header {"approved_by","dt","note"} as a signed contract
              in lines.json; block run_from_template when eps_header exists;
              have Noelle refuse templates without both eps_header and eps[]
    result:   succeeded — three-layer enforcement:
                1. run_from_template: blocked with message directing to build_from_eps
                2. build_from_eps: reads and preserves eps_header in output
                3. Noelle: fault issued, template refused if eps_header or eps[] absent
    revealed: the division of authority needs to be both in code AND in readmes.
              Documentation without enforcement is intention; enforcement without
              documentation is mystery. Both together make the rule real.

  - try:      consider pre-building map.json/path.json at template level
    result:   rejected on analysis — world coordinates depend on instance placement;
              inter-station segments are inherently network topology; adding a
              template-level map.json would duplicate lines.json at a different
              coordinate level with no benefit (work done twice, new format to maintain)
    revealed: lines.json IS already the template-level pre-build. formation maps
              (formations/{name}.json) ARE the CP-position pre-build. BUILD reads
              both and applies world transform. The architecture is already clean.
              The question was worth asking; the answer confirmed the existing design.

principle: Division of authority in JPods template topology:
             - Model designer: authors eps[] (physical topology claim) + signs with eps_header
             - build_from_eps: updates geometry only, preserves eps_header
             - run_from_template: blocked when eps_header exists (cannot overwrite approval)
             - Noelle: reads eps[], reports defects, refuses unsigned templates
             - BUILD: reads lines.json (local coords) + formation maps, applies world
               transform, emits map.json/path.json per network instance
           
           map.json and path.json are always per-network. lines.json is the
           template-level pre-build. These two levels must never be conflated.

           Noelle's refusal is the enforcement gate. A template without eps_header
           is physically incomplete — its topology is unverified, and Noelle
           cannot safely route through it regardless of what geometry exists.
