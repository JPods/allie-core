summary: SketchUp HtmlDialog camera follow-timer at 0.25s locks the viewport — use scene activation instead.
code: dispatch_server.rb:handle_camera_action

The follow_camera_tick approach (UI.start_timer 0.25s repeating, cam.set + view.camera = cam)
blocks SketchUp's viewport event loop. The screen locks and the user cannot interact.

The correct pattern: activate a saved SketchUp Scene (model.pages.selected = page).
Scenes are SketchUp's own viewport state machine — they transition smoothly, are user-reversible,
and do not hold any timer. The plugin names the scene by station friendly name or station ID;
the user creates one via View → Scenes → Add Scene.

Follow camera is removed. Scene activation is the new zoom_vehicle replacement.
