## What became clear

**map.json is a partial snapshot, not a complete network map.**

After extending trip.json to all 12 O-D pairs, Show FollowMe Overlay reported
30 of 46 segments as map gaps. Python inspection revealed:

1. map.json `segments` has only 3 of the 6 bidirectional inter-station connections —
   the 3 "outbound" ones (S049→S048, S050→S048, S051→S049). The 3 "inbound" ones
   (S048→S049, S048→S050, S049→S051) are absent because they were added to feature.json
   after the last map.json generation. map.json must be regenerated after every Build
   that adds new connections.

2. map.json `features` covers only S048. S049, S050, S051 intra-station geometry is
   entirely absent. This is not a code bug — the map.json generator scans SketchUp
   entities; if those entities weren't tagged correctly at generation time, they are
   silently absent.

3. trip.json uses dot notation (S048.gw_far_main); map.json feature cids use underscore
   (S048_gw_far_main). Silent key mismatch fixed with a regex translation in the lookup:
   `k.match(/\A(S\d+)_(gw_.+)\z/) → "#{m[1]}.#{m[2]}"`

4. Show Trip Path was wired to Nora vehicle IDs (`:nora_ids` source), not trip IDs.
   No vehicles placed = empty dropdown = user sees nothing. Fixed: `:trip_ids` source
   reads directly from trip.json; `show_trip_path_by_id` takes a trip ID, no vehicle required.

## What this means going forward

- After any Build that adds or removes connections, regenerate map.json.
- The "16 strips visible" result (S048 segments only) is correct given the current map.json.
  Full overlay requires a map.json rebuild covering all 4 stations.
- trip.json is the right source for "what Nora navigates." map.json is the right source
  for "where the geometry is." They are separate concerns.
