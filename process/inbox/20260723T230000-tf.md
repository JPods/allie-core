code: crash_harvester/harvest/arcgis_dot.py (ok_all config)

The pattern:
1. State DOT publishes an ArcGIS StoryMap or Dashboard for public viewing
2. Behind that app is a FeatureServer with the actual data
3. Search `site:arcgis.com "{state}" crash FeatureServer` to find it
4. Check the FeatureServer metadata for lat/lon, severity, fatality fields
5. Add to arcgis_dot.py CONFIGS and harvest

This should be automated — for every state in the "portal_required" list, Noelle/Allie should:
- Search for ArcGIS FeatureServer URLs behind dashboards
- Probe discovered URLs for field metadata
- Auto-add working endpoints to the harvester
- Report what was found and what still needs manual access

Apply to all 21 portal-required states. The Oklahoma success proves the pattern works.
