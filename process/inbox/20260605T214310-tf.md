## Station segment ID model

Dotted notation `s002.gw_cp_in_lead_0` is a compound lookup key that APPEARS
to be a namespaced path but is not. The correct mental model:

  Station `s002` is an object with an ordered array of track IDs:
    { "s002": ["gw_cp_out_lead_0", "gw_cp_out_0", ..., "gw_cp_in_lead_0", ...] }

  `gw_cp_in_lead_0` is a position in that array, not a child of `s002`.

The dotted compound key exists as a map lookup convenience (`s002.gw_cp_in_lead_0`)
but must never be used in JSON output as if it were a real hierarchical path.

Trip report defects should use structured fields, not dotted strings:
  WRONG:  { "type": "cp_gap", "before_seg": "s003.gw_platform_parking", ... }
  RIGHT:  { "type": "cp_gap", "station": "s003", "before_track": "gw_platform_parking", ... }

Inter-station segs (`seg_s001_cp0_s003_cp1`) are compound IDs, not dotted — they
are correct as flat strings because they name a connection, not a track within a station.

## Traffic circle approach curves

The approach curve check (check_approach_curves in noelle.rb) walks the first 12m of
inter-station guideways and flags curves below MIN_APPROACH_CURVE_RADIUS (cruise speed
minimum). This check MUST NOT run on guideways that connect to or from a traffic_circle
station — the traffic circle's own arc geometry determines the required entry radius,
and it is always tighter than the open-line minimum. Flagging these is always a false positive.

Fix: build tc_sids Set before the guideway loop; skip each probe whose endpoint_sid is
in tc_sids. Formation_id attribute on the station definition component is the authority.
