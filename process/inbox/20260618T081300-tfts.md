principle: Bezier curve station template tracks (gw_platform_in*, gw_lift_in,
           gw_lift_parking, etc.) have NO structural centerline edges in the model.
           Their centerline exists only in the jpods_path attribute — set by the
           connect tool for inter-station guideways, but NOT for intra-station
           template tracks.

           When jpods_path is absent or < 3 pts (poisoned by prior bad Extract),
           bbox fallback reads the nearest edge endpoint — which is one rail of the
           beam, ~250mm off-center. Adjacent tracks read opposite rails. Result:
           500mm gap that does not exist in the physical model.

           The midpoint between the two bbox endpoints is the correct centerline.
           Compute auto-snaps this gap when the pair is declared in designer.tracks
           (safe: no design intent for a real 500mm gap between successors).
