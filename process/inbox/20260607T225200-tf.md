summary: |
  Nora's path quality rule: no curve tighter than 3m diameter, no direction
  change within less than 3m of travel — except inside traffic_circle topology.

  A smooth JPods path has two geometric invariants:

  1. MINIMUM CURVE DIAMETER: 3m (radius 1.5m)
     No arc anywhere on the path may have a centerline radius < 1500mm.
     This matches MIN_STATION_ARC_RADIUS_MM in jpod_constants.rb.
     A curve tighter than this is physically unreachable without a jerk.
     Nora detects: centripetal acceleration spike where IMU lateral g-force
     implies radius < 1.5m. Record as physical.json defect, type "tight_curve".

  2. MINIMUM STRAIGHT BETWEEN DIRECTION CHANGES: 3m
     Any two consecutive direction changes must be separated by at least 3m
     of travel. A S-curve compressed into < 3m causes a jerk even if each
     individual arc is smooth.
     Nora detects: two heading-change events within < 3m encoder distance.
     Record as physical.json defect, type "direction_change_too_close".

  EXCEPTION: traffic_circle topology
     Traffic circles have tight arcs by design (inside radius = 1500mm = at
     the limit). Nora suppresses tight_curve defects when the current segment
     is flagged as traffic_circle in the trip plan. The 3m direction-change
     rule still applies between the circle entry and exit transitions.

  RECORDING:
     Both defect types go to physical.json per Axiom 9:
       { type: "tight_curve" | "direction_change_too_close",
         location_t: <fraction along segment>,
         severity: "minor" | "moderate" | "severe",
         description: "radius ~Xmm < 1500mm limit" | "direction change at Xm, previous at Ym (Zm gap)",
         logged_at: UTC,
         logged_by: "NORA_xxxx" }

  REPORTING:
     After logging, Nora publishes a fault message to Noelle:
       "path quality defect at [segment_id] location_t=[t] — map review requested"
     Noelle re-checks the junction coordinate and corrects if needed.

  These rules apply to the physical model (Pi Nora) and to the SketchUp
  proof (Show Route junction Z check catches the same class of defect geometrically).
  A jerk the physical pod feels is a map error. Fix the map; do not smooth the firmware.
