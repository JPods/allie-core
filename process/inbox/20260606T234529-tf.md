summary: JPods coordinates silently mix three frames of reference (structural/vehicle/vertical);
         every coordinate must declare which frame it is in or bugs are invisible.

code:    jpod_path_json.rb, jpod_animator.rb, jpod_constants.rb

## The three frames

| Frame | What | Key values |
|-------|------|-----------|
| Structural | Inside rail geometry — what SketchUp ArcCurves measure | radius = 1500mm (MIN_STATION_ARC_RADIUS_MM) |
| Vehicle path | Pod centerline — where the vehicle actually travels | radius = 1750mm (inside + BEAM_WIDTH/2 = 1500 + 250) |
| Vertical | Beam center (mid-beam), beam top face, or terrain surface | Z varies by ±250mm (BEAM_DEPTH/2) |

## Conversions

- Inside rail → centerline: scale radially outward by +BEAM_WIDTH/2 (250mm)
- Beam center → beam top: +BEAM_DEPTH/2 (250mm)
- Beam center → beam bottom: -BEAM_DEPTH/2 (250mm)
- Terrain → beam bottom: +CLEARANCE_HEIGHT (4600mm)
- Terrain → beam center: +CLEARANCE_HEIGHT + BEAM_DEPTH/2 (4850mm)
- Terrain → beam top: +CLEARANCE_HEIGHT + BEAM_DEPTH (5100mm)

## Where this caused bugs this session

- gw_uturn ribbon traced inside rail (1500mm) not centerline (1750mm) — ArcCurve IS the
  structural edge; extraction copied structural coords into vehicle path lookup
- followus ribbon Z-disconnected between tracks — some tracks stored pts at beam center,
  others at beam top; ribbon_z = max_z + lift looked correct sometimes, disconnected others
- arc radius check compared extracted radius to MIN_ARC_RADIUS_MM without confirming
  which frame the extracted radius was in

## The discipline required

Every function that produces or consumes coordinates must document its frame:
  pts in path.json → VEHICLE PATH frame, beam center Z
  pts in extracted.json → LOCAL TEMPLATE frame, beam center Z (after arc correction)
  CLEARANCE_HEIGHT → TERRAIN → BEAM BOTTOM edge (structural)
  ribbon_z → lifted above max beam top (display only, not vehicle path)

When a coordinate crosses a frame boundary, the conversion must be explicit and named.
"arc correction" is not enough — name the frames: "structural inside rail → vehicle centerline".
