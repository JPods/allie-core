summary: Name tracks by vehicle direction and the color standard becomes a live
         correctness check — wrong color = naming or topology defect, visible
         without running an animation.

code:    jpod_animator.rb:show_all_tracks — color logic block

detail:  show_all_tracks assigns color by parsing the track name:
           *_in   → RED  (inbound — vehicle arriving)
           *_out  → BLUE (outbound — vehicle departing)
           gw_c_* → YELLOW (ring/circle)
           other  → GREEN (platform, uturn, interior)

         Because JPods track names encode vehicle direction (gw_platform_in,
         gw_lift_in, gw_cp_out_lead, etc.), the color map is automatic. A
         designer opens model.skp → Show Formation Tracks and immediately sees:
           - Are all red tracks pointed into the station?
           - Are all blue tracks pointed out?
           - Is the ring yellow and continuous?
         Any track with the wrong color is a naming defect or direction error
         — the display catches what the animator would have silently propagated.

         This is Axiom 15 ("debug once, use many") applied to geometry checking:
         one visual pass replaces running multiple animation tests.
