detail: |
  Bill's principle: if a behavior can be turned on or off, encode it as one function
  with a parameter declaring intent. The caller states what state it wants; the function
  owns how to achieve it. Paired do_x/undo_x methods are always wrong — they split
  ownership, force callers to know implementation details, and create drift when one
  half is updated and the other isn't.

  Applied here: end_caps(model, install: true/false)
    install: true  — install caps on unconnected stubs, remove from connected CPs
    install: false — remove all caps (BUILD pre-step, before building tubes)
