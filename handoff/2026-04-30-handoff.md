### Isochrone — three-part fix

**Root cause**: simulation `trip_stats` only contains O-D pairs that received ≥1 randomly generated passenger; ~22% of pairs had 0 by Poisson chance, creating visible holes. Also, pods clustered at popular destinations after Sweep 1, leaving some stations pod-starved in Sweep 2.

**Fix 1 — Three-phase simulation** (`engine/simulation.py`):
- Phase 1 (Sweep 1): queue one passenger every-station-to-every-station; run until all complete
- Phase 2 (Sweep 2 + random demand concurrent): queue another full sweep; generate normal demand in parallel; stop when last Sweep 2 trip finishes
- Result: every reachable O-D pair gets ≥2 sample trips; Sweep 2 times reflect real congestion

**Fix 2 — Pod redistribution** (`engine/simulation.py`, `_arrive_at_line_end`):
- When a pod arrives at a station and isn't immediately re-dispatched, check if station is over `pods_per_station` quota and no passengers are waiting → return surplus pod to depot
- Prevents pod clustering after Sweep 1 that left some stations unable to dispatch in Sweep 2

**Fix 3 — Analytical Dijkstra fallback** (`gui/api.py`, `gui/static/timemap.js`):
- New endpoint `GET /api/network/travel_times?origin=<station_node_id>` — runs Dijkstra over network graph, returns travel minutes to all reachable stations
- `timemap.js` `_draw()` is now `async`; fetches analytical times after identifying boarding station
- Simulation medians (include overhead + congestion) used where available; analytical estimates fill any remaining gaps
- Station dot tooltips label: `(sim)` vs `(est)` to distinguish sources
