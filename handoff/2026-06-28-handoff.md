## Architecture Decisions

- **DataBrowser replaces dedicated admin pages** — GLAccount, Currency, Setting, Template, etc. all access through `/admin-wb?model=X`. Only user-facing models (Customer, Item, Invoice, Order, etc.) keep dedicated pages.
- **Shift-click any sidebar item → DataBrowser** — power user shortcut, no extra UI
- **Payment handles both AR and AP** — `type` field (received/disbursed) + `purchase` FK. No separate Expense model. One reconciliation flow, one GL posting path.
- **Layouts stored in Settings** — `purpose="workbench_fields"`, named views in `views[]` array with field lists + column widths
- **"initial" layout per model** — Alice's baseline for comparing user submissions
- **Fake records marked `metadata.health="fake"`** — prefixed "zz", sort to bottom, filterable
- **Submit for Bonus** — sidebar stub at `/submit-bonus`, layouts/components submitted via sync
